# DataFinderAgentOS 角色体系说明

## 文档定位
- 本文件记录当前项目中的角色定义、权限分配和用户-角色关系。
- 角色体系基于 RBAC（Role-Based Access Control）三级模型：用户 → 角色 → 功能。

---

## 一、系统预设角色

### 1. 系统管理员（admin）
| 属性 | 值 |
|------|-----|
| **角色ID** | 1 |
| **编码** | `admin` |
| **名称** | 系统管理员 |
| **描述** | 拥有系统所有权限 |
| **状态** | 启用 |
| **可编辑** | 否（系统固定角色） |
| **可删除** | 否 |
| **登录范围** | 管理侧-后台（`/admin/`） |
| **分配的功能** | 全部 15 个功能 |

### 2. 普通用户（user）
| 属性 | 值 |
|------|-----|
| **角色ID** | 2 |
| **编码** | `user` |
| **名称** | 普通用户 |
| **描述** | 只能登录前台用户侧 |
| **状态** | 启用 |
| **可编辑** | 是 |
| **可删除** | 是 |
| **登录范围** | 用户侧-前台（`/`） |
| **分配的功能** | 无（前台用户不使用后台功能机制） |

## 三、用户-角色关系

- 用户与角色是 **1 对 1** 关系（每个用户只能有一个角色）。
- 用户创建时默认分配"普通用户"角色（role_id=2）。
- 管理员可在用户管理页面修改用户的角色分配。
- 超管（admin 用户）固定为"系统管理员"角色，不可修改。

### 当前用户数据
| 用户名 | 角色 | 备注 |
|-------|------|------|
| admin | 系统管理员 | 超管，固定不可修改删除 |
| （其他用户） | 普通用户（默认） | 可通过管理侧修改角色 |

## 四、权限分配机制

### 角色-功能分配
- 通过 `role_functions` 表建立多对多关联。
- 在角色管理页面以 Layui 树组件形式呈现，支持多选。
- 系统管理员默认分配全部 15 个功能。

### 菜单映射
- 通过 `menus` 表将角色-功能映射为可排序的导航菜单。
- 当角色功能分配变更时，`run.py` 中的保障循环自动同步创建缺失的菜单记录。
- 用户最终看到的左侧菜单 = 根据用户角色查询 `menus` 表获得。

### 访问控制
- 管理侧页面通过 `@tornado.web.authenticated` 装饰器进行认证保护。
- 未登录用户访问 `/admin/` 下页面会被重定向到登录页。
- 当前 RBAC 粒度控制到页面级别（功能级别），未实现按钮级权限。

## 五、自定义角色

可从角色管理页面创建自定义角色，例如：

| 角色名 | 编码 | 描述 | 适用场景 |
|-------|------|------|---------|
| 菜单管理员 | menu_admin | 只能管理菜单 | 专门负责菜单配置的用户 |
| 采集员 | collector | 只能操作瞭望中心模块 | 专门负责数据采集的用户 |
| 数据分析师 | analyst | 模型引擎+数据仓库+大屏 | 专门负责数据分析和可视化的用户 |
