# DataFinderAgentOS 需求文档

## 系统背景
- **系统名称**：某政务智能瞭望与智能问数系统
- **系统类型**：Web B/S 结构的数据智能采集和智能问数系统
- **架构划分**：用户侧-前台 + 管理侧-后台
- **用户侧定位**：类 ChatGPT 的智能应用，智能问数 + 数据对话
- **用户侧功能**：用户登录、用户注册、用户对话、数字员工、@天气/@新闻/@音乐/@电影/@yummy、报表功能、智能问数、报告导出（PDF）
- **管理侧功能**：控制台、用户管理、功能管理、菜单管理、角色管理、瞭望采集、瞭源管理、数据仓库、深度采集、数字员工、模型引擎、数智大屏、舆情大屏

## 文档定位
- 本文件用于跟踪项目需求及完成状态，反映当前仓库的真实实现情况。
- 需求以用户侧-前台和管理侧-后台的模块划分为依据，标注完成状态。

---

## 一、已完成需求

### 1. 基础框架与认证
- **用户登录/退出** - 支持前后台双通道登录（`/login` 和 `/admin/login`），基于 secure_cookie 会话维持
- **首页仪表盘** - 后台首页展示统计卡片 + 系统概览
- **RBAC权限模型** - 用户-角色-功能三级权限体系，角色功能分配 + 菜单自动同步
- **控制台实时统计刷新** - 8项统计指标（用户/数据/任务/模型/瞭源/数字员工等），每30秒自动轮询更新
- **人脸识别登录** - 前端模板 `face_login.html`（摄像头面部捕捉），后端待对接

### 2. 用户管理（管理侧）
- 用户 CRUD，分页搜索，状态管理
- 超管保护机制（不可修改/删除超管账号）
- 密码编辑（支持在用户表单中重置密码）
- 管理员标识（admin角色不可删除）

### 3. 功能管理（管理侧）
- 功能 CRUD（支持父子层级树结构）
- 启用/禁用切换
- 级别展示（图标+层级缩进显示功能级别）

### 4. 菜单管理（管理侧）
- 菜单 CRUD（按角色配置菜单）
- 排序管理 + 菜单预览
- 自动同步：角色功能分配变更时自动更新菜单
- **图标图形化展示**：功能图标以 `<i class="layui-icon ...">` 形式展示

### 5. 角色管理（管理侧）
- 角色 CRUD，角色-功能分配树
- 管理员保护，菜单自动生成
- **编辑弹窗**：完整实现编辑弹窗（AJAX获取→弹窗回填→保存更新），支持状态修改

### 6. 瞭望采集（管理侧）
- **瞭源管理** - 采集源CRUD，支持URL模板、请求头、分页参数、解析器选择
- **瞭望采集执行** - 按关键词+分页规则批量爬取
- **采集结果橱窗展示** - 卡片式布局，全选/全不选、批量保存到数据仓库（去重upsert）
- **分页参数** - 步长可编辑并同步瞭源配置，手动选择采集页数

### 7. 数据仓库（管理侧）
- **数据仓库管理** - 列表/详情/删除/批量删除/搜索
- **采集数据存储** - 瞭望采集结果保存到数据仓库（去重upsert）
- **深度采集标识** - 列表中标识是否已深度采集
- **批量操作** - 支持批量选中后删除/深度采集

### 8. 深度采集（数据仓库子功能）
- 单条/批量深度采集（通过数字员工-采集专员调度）
- crawl4ai 网页爬虫集成（Playwright Chromium 引擎）
- 任务监控面板（悬浮窗，实时进度/步骤/日志/员工详情）
- 采集结果持久化（deep_collect_data 表）
- 更新采集（覆盖旧数据）、已采集数据查看

### 9. 瞭源管理（管理侧）
- 采集源 CRUD + 规则配置
- 支持多解析器类型：html（百度新闻）、json_api（Hacker News）、rss（36氪）
- 3个预设瞭源（百度新闻 + Hacker News + 36氪）

### 10. 接口管理（管理侧）
- 接口 CRUD（名称/编码/请求方法/URL模板/请求头/参数模板/响应模板/状态/描述）
- 搜索（名称/编码）、分页（20条/页）
- 内置4个预设接口（聚合新闻、TMDB电影、iTunes音乐、和风天气）
- 与数字员工联动（"关联接口"下拉选择，自动填充API配置）

### 11. 技能管理（管理侧）
- 技能 CRUD（名称/编码/类型/实现方式/配置/描述），分页搜索
- 三种实现方式：Python函数（importlib）/ API调用（HTTP）/ Prompt模板（变量替换）
- 内置1个预设技能（"今日热点摘要" --- 调用聚合新闻API）
- 技能执行引擎 SkillExecutor（支持链式传递）
- 与数字员工联动（LLM型可多选关联技能，保存到 skills 字段）

### 12. 模型引擎（管理侧）
- 模型 CRUD，分类管理（text/image/audio/video/multimodal/embedding）
- 默认模型设置，SSE流式AI对话
- Token统计，对话参数调节（Temperature/Top-P/Max-Tokens）
- Markdown渲染（marked.js）
- 网格布局（1行3列，每页6条）

### 13. 数字员工（管理侧）
- 数字员工 CRUD，支持两种类型：
  - LLM型（关联模型+系统提示词+技能+crawl4ai）
  - API型（自定义HTTP端点）
- MD文件上传（`data/dgUser/` 目录）
- 数字员工测试功能
- 智能中枢菜单组织（模型引擎+数字员工归入统一父菜单）

### 14. 数智大屏（管理侧，v0.3新增）
- `/admin/data-screen` 路由 + 4个统计API
- ECharts可视化：统计面板、词云、趋势折线图、来源柱状图、桑基图
- 数据实时刷新

### 15. 舆情大屏（管理侧，v0.3新增）
- `/admin/opinion-screen` 路由 + 7个API
- 实时舆情监控、预警级别标识（红/橙/黄）
- AI智能分析、敏感词管理（CRUD）、情感分析
- 人工反馈处理闭环

### 16. 多模态API（v0.3新增）
- 图片生成（ImageGenHandler）：优先配置模型 → OpenAI DALL-E → Pollinations.ai回退
- 视频生成（VideoGenHandler）：优先配置模型 → Pollinations.ai回退
- 前台路由：`/api/user/image-gen`，`/api/user/video-gen`

### 17. PDF导出（v0.3新增）
- UserExportPdfApiHandler：按 conversation_ids 导出对话为PDF
- reportlab + Windows微软雅黑中文字体
- 用户数据隔离（仅能导出自己的对话）

### 18. 解析器体系（v0.3新增）
- 数据源解析抽离为独立 `parsers/` 包
- 百度新闻HTML解析器、JSON API通用解析器、RSS订阅源解析器

### 19. 个人信息管理
- 查看个人信息，修改当前用户密码

### 20. 用户注册（用户侧）
- 注册页面 `register.html`（双栏品牌布局）
- 注册API（用户名长度/密码长度/重复密码校验）
- 注册后自动分配普通用户角色
- 用户名唯一性检查

### 21. 用户对话（用户侧）
- chat.html 五区布局（LOGO+标题 / 模型切换 / 历史列表 / 对话区 / 输入区）
- SSE流式对话输出，Markdown渲染（markdown-it）
- 聊天气泡（左AI右用户），对话历史本地存储
- 欢迎页面和快捷建议，响应时间与Token显示
- 意图识别与数据查询（智能问数 + 数据库反馈）

### 22. 用户侧数字员工与@技能命令
- @菜单激活数字员工列表
- LLM型员工优先使用关联模型，API型直接调用配HTTP端点
- 消息气泡显示当前激活的员工名称标签
- @天气、@新闻（聚合数据API+百度兜底）、@随机音乐（iTunes API+本地精选）、@电影（TMDB API+本地库+自然语言关键词提取）、@yummy 技能调用
- 内置 Mock API 型员工通过 `_call_builtin_mock_api()` 直调函数实现，零网络开销

### 23. 用户侧快捷功能
- /菜单激活快捷功能（生成报表、生成图表、数据分析、导出报告）

### 24. 用户侧界面统一
- 登录/注册输入框样式与后台管理侧完全一致
- 后台Logo文字显示完整无遮挡（line-height/width CSS修复）
- 智能中枢图标更换为 `layui-icon-bot`，功能/菜单管理同步更新

### 25. 主题切换（v0.3新增）
- 亮色/暗色主题切换（`theme.css`）
- 后台管理侧 `admin.css` 专用样式

### 26. 数据迁移（v0.3新增）
- 旧版 `big_screen` → `data_screen` 自动迁移
- 旧版 `layui-icon-engine` → `layui-icon-bot` 自动迁移
- 舆情大屏图标自动修正
